# 1. Record architecture decisions

- Status: accepted
- Date: 2026-07-06

## Context

We want a lightweight, durable record of the significant architectural choices in
MuscleApp, both to keep the team aligned and to support the thesis report.

## Decision

We will use Architecture Decision Records (ADRs), one Markdown file per decision in
`docs/adr/`, numbered sequentially. Each ADR states context, decision and consequences.

## Consequences

- Decisions are traceable and reviewable in pull requests.
- New contributors can understand *why*, not just *what*.
